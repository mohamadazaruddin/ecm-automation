A solid “efficiency” metric here is a first-pass success rate: the share of cases that go forward vs. needing pushback.
Recommended efficiency metrics
1) Basic Efficiency (First-pass rate)
Efficiency = Fwd Push / (Fwd Push + Material Pushback + Critical Pushback)

1.0 = everything goes forward; 0.0 = everything is pushback

Also track (so you don’t hide quality):

Pushback Rate = (Material + Critical) / Total
Critical Share = Critical / Total (useful if critical is more costly)

2) Weighted Efficiency (if critical is “worse”)
If critical pushbacks are more severe than material:
Weighted Efficiency = Fwd / (Fwd + 1×Material + 2×Critical)
(You can tune 2→3, etc., based on business impact.)
3) Complexity-adjusted (optional, if High cases should “count more”)
Compute per complexity, then roll up with weights, e.g.:

Low=1, Medium=1.5, High=2 (example)
This avoids penalizing an analyst who gets harder work.

let
    Source = #"Previous Step",
    Grouped = Table.Group(
        Source,
        {"Analyst Name"},
        {
            {"Fwd", each List.Sum([fwd Push]), type number},
            {"Material", each List.Sum([material pushback]), type number},
            {"Critical", each List.Sum([critical pushback]), type number}
        }
    ),
    AddTotal = Table.AddColumn(Grouped, "Total", each [Fwd] + [Material] + [Critical], type number),
    AddEfficiency = Table.AddColumn(AddTotal, "Efficiency", each if [Total] = 0 then null else [Fwd] / [Total], type number),
    AddWeightedEfficiency = Table.AddColumn(
        AddEfficiency,
        "Weighted Efficiency",
        each
            let denom = [Fwd] + [Material] + 2 * [Critical]
            in if denom = 0 then null else [Fwd] / denom,
        type number
    )
in
    AddWeightedEfficiency
